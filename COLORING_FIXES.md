# Исправления для раздела раскрасок

## Изменения

### 1. Исправлена ошибка 500 при загрузке отдельной раскраски
- Исправлен SQL запрос в `/api/coloring-sections/:sectionSlug/:pageSlug`
- Теперь сначала находится раздел по slug, затем раскраска по section_id
- Корректные сообщения об ошибках: "Раздел не найден", "Раскраска не найдена"

### 2. Загрузка изображений разделов с устройства
- Добавлена возможность загрузки изображений с устройства вместо URL
- Поддерживаемые форматы: JPEG, PNG, GIF, WebP (до 5 МБ)
- Изображения сохраняются в `server/files/coloring/`
- Автоматическое переименование файлов по slug раздела
- **Исправлено**: превью обновляется с кэш-параметром для отображения новых изображений

### 3. Управление разделами в админ-панели
- Список всех разделов с превью изображений
- **Новое**: кнопки для изменения порядка разделов (вверх/вниз)
- Кнопка загрузки изображения для каждого раздела
- Кнопка удаления раздела
- Отображение текущего порядка раздела

### 4. Сортировка разделов
- Добавлено поле `sort_order` в таблицу `coloring_sections`
- Разделы сортируются по `sort_order ASC, created_at DESC`
- Миграция применяется автоматически при запуске сервера

## Автоматическая миграция

**Миграция применяется автоматически при перезапуске сервера!**

При запуске сервера выполняются:
1. Создание таблицы `coloring_sections` (если не существует)
2. Добавление поля `image_url` (если отсутствует)
3. Добавление поля `sort_order` (если отсутствует)
4. Заполнение тестовыми разделами (если таблица пуста)
5. Присвоение URL изображений существующим разделам

Вам **не нужно** вручную применять SQL-миграции.

## API Endpoints

### Получение списка разделов
```
GET /api/coloring-sections
```
Возвращает разделы, отсортированные по `sort_order`

### Изменение порядка разделов (admin)
```
PUT /api/admin/coloring-sections/reorder
Content-Type: application/json
Authorization: Bearer <token>

{
  "sections": [
    { "id": 1, "sortOrder": 0 },
    { "id": 2, "sortOrder": 1 },
    { "id": 3, "sortOrder": 2 }
  ]
}
```

### Загрузка изображения в раздел (admin)
```
POST /api/admin/coloring-sections/:id/image
Content-Type: multipart/form-data
Authorization: Bearer <token>

image: <file>
```
